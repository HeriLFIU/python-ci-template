# Semantic Versioning

## Explanation

Semantic Versioning also known as SemVer is a specification which tells you how version numbers should be written and managed.

Ex. v1.0.1 is composed of the major version number, the minor version number, and the patch version number (vMajor.Minor.Patch).

You increment the major version number when you make a breaking change, and users cannot safely upgrade to it without expecting errors. You increment a minor version number when you add a new feature. You increment the patch version number when you make a small change or a bug fix.
